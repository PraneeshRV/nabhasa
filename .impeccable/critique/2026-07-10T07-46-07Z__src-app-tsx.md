## Anti-Patterns Verdict

**Not AI slop.** Register coherence is auteur-grade: one accent enforced across shaders AND UI, real physics as content, lore mapping 1:1 to build tasks. Deterministic scan agrees: **detect.mjs exit 0, zero findings** across index.html + src. Runtime overlay scan of the static page: 7 warnings, mostly benign in context (pulsar-dot glow, root overflow clip = false positives; 3 all-caps table row headers borderline; Space Grotesk flagged overused-font — identity-preservation applies, it's committed brand canon).

**But**: demo-ware seams sit at exactly the product-critical points — a literal editorial placeholder in the Contact block, a "content parity" claim the static page does not fulfill, and a mission system with no entry-point UI. Unfinished is what AI slop and rushed human work have in common.

## Overall Impression

The craft layer (preloader, shaders, fail-recovery engineering, non-modal panels) is exceptional; the product layer is unfinished. The single biggest opportunity: the experience's climax and its most likely visitor path both currently deliver zero portfolio value — fix the three P0s and this goes from tech demo to portfolio.

## What's Working

1. **Collapse preloader** (CollapsePreloader.tsx): real load signals only, time floor + 9s safety timeout, audio unlock inside the gesture, focus moved to primary button. Branded loading done right.
2. **Non-modal diegetic panels** (ApproachPanel + approach.css): pointer-events-none root, ESC/fly-away dismissal with per-world memory, documented AA contrast, reduced-motion honored. B confirms: static page has correct heading order, skip link, captioned+scoped tables, ~6.6:1 contrast, zero console errors on both routes.
3. **Failure-recovery engineering** (Craft.tsx): body frozen under overlays, forces cleared on respawn, respawn at the failed mission's source. Someone walked the fail path repeatedly.

## Priority Issues

**[P0] The mission system is invisible — the core loop cannot be discovered.**
Why: `offered` status has no UI in the codebase; keyboard hints omit KeyC; beacons render only when active. The 5-mission spine ending at Contact is reachable only by reading source.
Fix: offer toast when `offered` ("EMBERLIGHT OUTBOUND · C TO ACCEPT"), add `C ACCEPT` to hints, pulse destination beacon during offer.
Command: /impeccable onboard

**[P0] The Contact endpoint contains no contact method.**
Why: portfolio.json Contact ends in an editorial placeholder; ApproachPanel renders blocks as plain `<p>` — no anchor rendering path exists. PRODUCT.md success metric is "gets in touch".
Fix: real links (mailto/GitHub/LinkedIn) in JSON + a `links` field on Block rendered as focusable anchors.
Command: /impeccable harden

**[P0] Content parity claimed but absent: static tier and mobile film contain zero portfolio content.**
Why: StaticExperience renders astrophysics only — no About/Projects/Contact, not even the name. B's heading inventory corroborates: sections are System status / star / physics / moments / planets — no portfolio sections. Mobile FlythroughFilm likewise anonymous. Mobile recruiters = most resume clicks.
Fix: render all five portfolio.json sections on the static page; mobile film gets section cards or a persistent "read the portfolio" link.
Command: /impeccable harden

**[P1] Reading a panel is a race against orbital mechanics.**
Why: Praesidium moves ~11.6 wu/s vs 60 wu radius; a braked reader loses the About panel in 5–10s mid-paragraph. The finale's `pinned` flag proves the team knows proximity-close is wrong for reading. Related: delivery beacons are fixed snapshots while planets orbit — you deliver "to the garden" in empty space.
Fix: once open, panel stays until ESC or active fly-away (distance increasing AND thrusting); consider snapping beacons to live positions (already published via getPlanetPositions()).
Command: /impeccable polish

**[P1] No non-flight path to content + no onboarding beat on desktop.**
Why: recruiter must master decoupled 6DOF to reach any fact; CraftCursor swells suggestively but steers nothing; first 60s stacks four learning tasks (cognitive-load: 4 checklist failures = high).
Fix: "VIEW AS TEXT" affordance on live tier; control map on the ENGAGE screen's dead time; first-run objective line ("FLY TO THE GARDEN — 260 KM SUNWARD").
Command: /impeccable onboard

**[P2] Fuel/fail loop lacks warning and exit.**
Fix: fuel row pulses <20%; bind + hint mission abandon (reducer already supports it).
Command: /impeccable harden

## Persona Red Flags

**Jordan (recruiter, 2 min, first-timer):** mobile → beautiful anonymous film, zero facts, zero name, zero contact — worst outcome on the most likely path. Desktop → jargon telemetry, no goal, clock expires before Praesidium. Even success is fragile (panel orbits away mid-read).

**Alex (power user):** will love the slingshot economy — if they ever find the game (KeyC discoverable only by mashing keys inside an invisible 40wu sphere). Distance-only nav = hunting a beacon that may be behind the camera. No mission select/replay; share-cancel forces a PNG download anyway.

**Sam (accessibility):** designated static route is missing the entire portfolio. ApproachPanel overflow-y:auto with no focusable element — keyboard users can't scroll it. "ESC · FLY AWAY TO DISMISS" is 10px --ui-dim (~2:1, fails AA). No mute toggle after one-shot sound choice. Telemetry's promised screen-reader summary region doesn't exist in the live DOM.
